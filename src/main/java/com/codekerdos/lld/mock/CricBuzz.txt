Design the backend models and services for a live cricket scoring application. The system needs to track the state of a cricket match ball-by-ball, update the scorecard in real-time, and serve this data to front-end clients.
Core Requirements
Match Tracking: The system must model a cricket match involving two teams, a specific format (e.g., T20, ODI), and track the current state of the match (Scheduled, In Progress, Completed, Abandoned).
Ball-by-Ball Granularity: Track every ball bowled in an over. A ball can result in normal runs, extras (Wide, No-Ball, Leg-Bye), and wickets (Bowled, Caught, Run Out, etc.).
Dynamic Scorecard: Maintain a live scorecard that instantly reflects:
Total team score and wickets.
Current Run Rate (CRR).
The current striker and non-striker with their individual scores.
The current bowler's figures (Overs, Maidens, Runs, Wickets).
Real-Time Updates : The system must push live score updates and ball-by-ball commentary to subscribed clients or dashboards whenever a ball is bowled, without the client needing to constantly poll the database.
